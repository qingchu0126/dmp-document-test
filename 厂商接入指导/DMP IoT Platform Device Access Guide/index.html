    DMP IoT Platform Device Access Guide

    This article mainly introduces the core process of device access to the DMP IoT platform and explains in detail the preparations that manufacturers must complete before connecting to the platform. If device developers need to connect IoT devices to the platform, please follow the steps described in the document one by one.

    Terminology Explanation

    • Please refer to Terminology Explanation.

    Preparations Required by Manufacturers Before Integration

    Provide Product Information

    • Provide the necessary information such as the product name, product category, product model, and communication protocol of the devices to be produced.
    • Jiuan will create the product in the management back-end based on the information provided by the manufacturer.

    Provide Product Model

    • Manufacturers need to specify the standard functions of the device model. Please refer to the attached Product Model Standard Functions Template.xlsx.
    • Select the model functions included in the device in the attached Excel file and send it back to Jiuan.
    • If the model functions of your device are not listed in the attachment or the existing functions do not meet your needs, please contact Jiuan for additions.
    • Jiuan staff will import the product model into the management back-end.

    Apply for Device Authorization Code

    • Determine the number of devices to be produced, and Jiuan will provide the corresponding number of device authorization codes.

    Apply for Device SDK

    • Manufacturers need to provide the SDK compilation chain to Jiuan. Examples of compilation chains are as follows:

    Example 1: arm-anykav500-linux-uclibcgnueabi

    Example 2: arm-anycloud-linux-uclibcgnueabi

    • Select the optional functions in the SDK. Optional functions include RTMP push, ONVIF, RTSP service, media services (including HLS, FLV, RTMP). Manufacturers should choose based on actual needs. If the manufacturer already has the above functions, they do not need to be selected again.
    • Jiuan will generate the corresponding device SDK based on the information provided by the manufacturer and provide it to the manufacturer.

    Access Steps

    Step 1: Platform Creates Product

    This step is performed by Jiuan staff.

    Create Product

    • The platform creates a device access type product in the system based on the information provided by the manufacturer.
    • After creating the product, the platform will generate a unique identifier (PID) for it.

    Step 2: Function Definition

    This step is performed by Jiuan staff.

    Define Model

    • Define the function points of the device under the product. A unified model helps standardize the data format between the device and the platform, simplifying device and application development.

    Associate Firmware

    • Define the firmware used by the device under the product.

    Step 3: Product Configuration

    This step is performed by Jiuan staff.

    Function Settings

    • Define the functions that users can control on the APP device settings panel.

    Product Display

    • Define the icon, product name, and product model displayed on the APP.

    Wizard Configuration

    • Define the device network configuration process on the APP.

    Step 4: Product Pre-release

    This step is performed by Jiuan staff.

    Product Pre-release

    • Pre-release refers to a pre-test and trial process before the official release of the product.

    Generate Authorization Code

    • The authorization code includes the product PID, device UUID, device key, QR code identifier, and device addressing domain.

    Step 5: Device Development

    This step needs to be completed by the manufacturer.

    Integrate Device SDK

    • Based on the function points defined by the manufacturer, the platform (Jiuan) provides the device SDK to the manufacturer. The manufacturer needs to complete the development work on the device side.

    Burn Authorization Code

    • The manufacturer needs to burn the device authorization code provided by the platform (Jiuan) into the device. Each device corresponds to a unique set of authorization codes. Please note that a set of authorization codes can only be burned into one device and cannot be reused.

    Device Debugging

    • Based on the defined model, perform data interaction debugging through the device debugging function, view real-time logs of the device, and conduct pre-production functional data verification.

    Step 6: Product Release

    This step is performed by Jiuan staff.

    Mass Production Release

    • For developed devices, mass production release can be carried out directly. In principle, the product cannot be modified after mass production release.
